### 步骤 3: Git 操作

```bash

git status

# 添加所有更改
git add .

# 提交
git commit -m "重构：分离 botA 和 botB，添加 GitHub Actions 自动部署"

# 推送到 GitHub
git push origin main
```

### 步骤 4: 关于 CRLF 警告

这些警告是**正常的**，不影响功能：
- Git 会自动将 Windows 行尾（CRLF）转换为 Unix 行尾（LF）
- 这是为了确保代码在不同操作系统上都能正常运行

如果想消除这些警告，可以配置 Git：

```bash
# 配置 Git 自动处理行尾转换
git config core.autocrlf true

# 或者如果只想在提交时转换（推荐）
git config core.autocrlf input
```

然后重新添加文件：
```bash
git add --renormalize .
git commit -m "规范化行尾格式"
```
